# PaddleOCRとAI Builder比較検証計画書

作成日時: 2026-03-03 23:33:04 +09:00
作成者: Codex + GPT-5

## 1. 文書の目的

本書は、ローカル OCR 案の本命候補である `PaddleOCR / PP-Structure` と、M365 ネイティブの `AI Builder document processing` を比較し、今回の帳票にどちらが適するかを検証するための計画書である。

## 2. 比較対象

### 比較対象A

- PaddleOCR
- 想定構成: PP-OCR + PP-Structure
- 実行場所: 社内ローカルサーバまたは VM

### 比較対象B

- AI Builder document processing
- 実行場所: Power Platform

## 3. 比較の目的

- PDF 帳票から必要項目をどこまで安定して抽出できるか確認する
- レイアウトゆれ、結合セル、品質ばらつきへの強さを比較する
- `年 6,000 件` 規模での実運用コストを比較する
- 正式指示値に使う前提で、危険な誤抽出をどれだけ防げるかを比較する

## 4. 対象項目

### Phase 1 の対象項目

- 機種
- 積載
- 停止階
- 昇降行程

### Phase 2 の追加候補

- 号機
- 現場識別子
- かご寸法系
- マニュアル選定条件
- 図面選定条件

## 5. 対象データセット

### 想定収集量

| 区分 | 件数仮定 |
| --- | --- |
| 学習 / 調整用 | 60 件 |
| 評価用 | 30 件 |
| 合計 | 90 件 |

### データの偏りを避ける観点

- 帳票パターン別
- スキャン品質別
- 結合セル多寡
- 表記ゆれ有無
- 複数ページ構成

## 6. 比較観点

| 観点 | PaddleOCR | AI Builder |
| --- | --- | --- |
| 初期ライセンス費 | 低い | ある |
| ページ従量課金 | なし | あり |
| ローカル実行 | 可能 | 不可 |
| M365 親和性 | 中 | 高い |
| レイアウト解析 | ある | ある |
| 表認識 | ある | 限定条件あり |
| 運用保守の自前責任 | 高い | 中 |
| 学習 / 調整のしやすさ | 高いが自前 | Power Platform 上で完結しやすい |

## 7. 評価指標

### 項目別指標

| 指標 | 定義 |
| --- | --- |
| 完全一致率 | 正解値と完全一致した割合 |
| 正規化一致率 | 単位や表記ゆれ正規化後に一致した割合 |
| 空欄率 | 抽出できなかった割合 |
| 危険誤抽出率 | 不正値を正しいように返した割合 |

### 帳票単位指標

| 指標 | 定義 |
| --- | --- |
| 4項目完全一致率 | 4項目すべて正しい割合 |
| 自動返信候補率 | そのまま自動処理に回せる割合 |
| 確認送り率 | 人手確認が必要と判定された割合 |
| 処理時間 | 1件あたり平均時間 |

### 運用指標

| 指標 | 定義 |
| --- | --- |
| 調整工数 | モデル設定、辞書設定、再学習の工数 |
| 障害対応難度 | 運用時の切り分けのしやすさ |
| 再現性 | 同条件で安定した結果が出るか |

## 8. 比較手順

### Step 1: データ準備

1. 代表帳票を収集
2. 正解ラベルを付与
3. 学習 / 評価に分離

### Step 2: PaddleOCR 側試験

1. PDF を画像化または直接入力
2. PP-Structure でレイアウト解析
3. OCR 結果から項目抽出ロジックを実装
4. 正解と比較

### Step 3: AI Builder 側試験

1. General documents モデルを作成
2. 4 項目をタグ付け
3. 推論結果を取得
4. 正解と比較

### Step 4: 運用観点比較

1. 失敗例を分類
2. 再調整のしやすさを比較
3. 年間費用を比較

### Step 5: 判断

1. 精度
2. 危険誤抽出率
3. 保守性
4. 年間追加費用

で採用判断する

## 10. 期待される仮説

### 仮説1

- `AI Builder` は M365 統合と PoC 立ち上がりが速い

### 仮説2

- `PaddleOCR` はライセンス費を抑えやすいが、項目抽出の後処理実装が重い

### 仮説3

- `結合セルや表構造が複雑` な帳票では、どちらも単純 OCR だけでは足りず、業務ロジック側で補助が必要

## 11. 年間コスト比較の仮置き

### 前提

- 6,000 件 / 年
- 5 ページ / 件

| 案 | 年間追加費用目安 | 備考 |
| --- | --- | --- |
| PaddleOCR | 0 〜 1,500 ドル相当 | サーバ流用前提、人件費除く |
| AI Builder | 約 2,580 〜 2,760 ドル | Credits + Premium 1〜2席想定 |

## 12. Go / No-Go 条件

### PaddleOCR 採用条件

- 項目別完全一致率が AI Builder と同等か、わずかに劣る程度
- 危険誤抽出率を十分に抑えられる
- 社内で保守可能

### AI Builder 採用条件

- 精度差が明確に高い
- Power Platform 側で運用を閉じたい
- 年間追加費用が許容範囲

## 13. 推奨

- まず比較検証対象は `PaddleOCR` と `AI Builder` の 2 本に絞る
- `NDLOCR-Lite` は比較対象から外すか、参考試験にとどめる
- 業務上の本命判断は、精度だけでなく `危険な誤抽出率` と `保守責任` を含めて行う
