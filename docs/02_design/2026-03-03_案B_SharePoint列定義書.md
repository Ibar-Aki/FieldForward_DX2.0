# 案B SharePoint列定義書

作成日時: 2026-03-03 23:33:04 +09:00
作成者: Codex + GPT-5

## 1. 文書の目的

本書は、自動化案Bを前提として、SharePoint Online に配置するライブラリとリストの列定義を具体化するものである。実装担当、業務担当、運用担当が同じ前提で話せるように、表示名、内部用途、型、必須性、例、注意点を整理する。

## 2. 前提

- 共有メールボックスで受信したメールを Power Automate で処理する
- 送付元メールアドレスから現場または案件候補を特定する
- 既存 Excel 計算ツールを Excel Online / Office Scripts で実行する
- 結果帳票と必要資料を返信する
- 正式指示値を返すため、監査証跡と版管理を残す

## 3. SharePoint 構成

### サイト候補

- サイト名: `FieldForward_DX2`
- サイト用途: 帳票原本、案件マスタ、資料マスタ、確認キュー、返信履歴の一元管理

### 作成対象

1. ドキュメント ライブラリ `受付原本`
2. リスト `送付元案件マスタ`
3. リスト `資料マスタ`
4. リスト `確認キュー`
5. ドキュメント ライブラリ `返信結果`
6. ドキュメント ライブラリ `計算テンプレート`

## 10. インデックス推奨列

検索性能とフロー安定化のため、以下にインデックスを推奨する。

- `送付元案件マスタ.送付元メール`
- `送付元案件マスタ.案件ID`
- `資料マスタ.資料ID`
- `資料マスタ.返却対象フラグ`
- `資料マスタ.承認済フラグ`
- `確認キュー.キュー状態`
- `確認キュー.案件ID`

## 11. 命名ルール案

- 原本ID: `SRC-YYYYMMDD-NNNN`
- キューID: `Q-YYYYMMDD-NNNN`
- 返信ID: `RES-YYYYMMDD-NNNN`
- 案件ID: 既存業務コードを流用
- 現場ID: 既存業務コードを流用

## 12. 実装前に決めるべきこと

1. 送付元メールが 1 件に対して複数案件を持ち得るか
2. 既定値と PDF 抽出値が矛盾した時の優先順位
3. 現場専用図面のアップロード責任者
4. 保持期限と削除方針
5. 結果帳票を Excel と PDF のどちらで正本とするか
