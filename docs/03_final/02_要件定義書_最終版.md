## 1. 文書の目的

本書は、建設現場におけるエレベータ据付支援のためのメール起点共通基盤について、機能要件・非機能要件・保守運用要件を体系的に整理した要件定義書である。

---

## 2. システム目的

- 現場での手計算を減らす
- 複数資料の探索負荷を減らす
- 正式な据付指示値を標準ルールで返せるようにする
- 今後、入力項目・計算ロジック・返却物を追加できる共通基盤とする

---

## 3. 想定利用者

| 利用者 | 主な役割 |
| --- | --- |
| 現場作業者 | 仕様票PDFをメール送信し、返信帳票を据付作業に使用する |
| 工事管理担当 | 案件管理、送付元マスタ更新、確認キュー運用を行う |
| 設計・技術部門 | 計算ロジック保守、資料更新、Excel正本管理を担う |
| 品質・安全部門 | 正式指示値の品質統制、監査ログ確認を行う |
| システム運用担当 | Power Automateフロー保守、障害対応を担う |

---

## 4. 対象範囲

### 4.1 対象

- メールでの帳票受付
- 添付 PDF の保管と解析
- 仕様値の抽出と正規化
- 計算ロジック実行
- 関連資料の特定
- 現場向け結果帳票の生成
- 返信メール送信
- 処理履歴の記録

### 4.2 非対象

- 現場向け常時オンライン Web UI
- CAD データの編集
- 原図作成
- 現時点での全仕様項目（30項目）対応
- 多言語対応
- 据付以外の業務への横展開

## 5. 前提条件

### 5.1 業務前提

| 前提 | 内容 |
| --- | --- |
| 入力形式 | PDF 形式の仕様票を添付してメール送信 |
| 帳票特性 | 一定の記載ルールあり、フォーマット完全統一ではない。Excel由来の結合セルを含む |
| 出力値の位置づけ | 参考値ではなく正式な据付指示値 |
| 利用者規模 | 約500人 |
| 年間処理件数 | 約6,000件（日平均24件、ピーク30件/日） |
| メール単位 | 基本は 1メール 1案件 1PDF |
| 案件特定 | 送付元メールアドレスと案件の対応は `1メールアドレス = 1案件` |
| 同日複数送付 | 同一現場からの同日複数メールは各々別案件として扱う |
| 本文依存 | 作業長はメール本文への自由記述を前提としない |

### 5.2 既存 Excel 計算ツール前提

| 前提 | 内容 |
| --- | --- |
| ファイル数 | 複数ファイル |
| 入出力 | 入力セルと出力セルは固定 |
| VBA | 使用している |
| 外部参照 | なし |
| 人手補正 | なし |

### 5.3 セキュリティ前提

- 個人情報・機微情報は含まれない
- 添付PDFのセキュリティレベルが最も重要
- 送信元制御は厳格には求めていない

### 5.4 フェーズ適用前提

| フェーズ | 主な成立条件 |
| --- | --- |
| PoC | 送付元マスタ、既存 Excel、簡易結果帳票で成立性を確認 |
| 先行導入 | 送付元マスタ整備率向上、確認キュー運用確立 |
| 拡張導入 | OCR や PDF 項目抽出を追加検討 |

> 補足: PoC と先行導入では、送付元マスタを主軸に運用し、`FR-08` から `FR-15` の帳票解析 / 正規化機能は将来拡張要件として扱う。

## 6. 業務フロー要件

```
 1. 利用者が特定メールアドレスに仕様票 PDF を送る
 2. システムがメールと添付ファイルを受信する
 3. システムが帳票種別判定と項目抽出を行う
 4. システムが抽出値を内部共通キーへ正規化する
 5. システムが計算ロジックと資料特定ロジックを実行する
 6. システムが結果帳票を生成する
 7. 条件に応じて自動返信または確認フローへ回す
 8. 利用者へ結果帳票と関連資料を返信する
 9. 処理履歴・使用ルール・入力値・出力値を記録する
10. 添付PDFが仕様票以外と判定された場合はエラー返信する
```

## 11. サンプル計算要件

### 11.1 対象

機種・積載・停止階・昇降行程から、ガイドレール用ブラケット個数を算出する。

### 11.2 入力項目

| 項目 | データ型 | 備考 |
| --- | --- | --- |
| 機種 | 選択肢（標準/オーダー） | 機種係数に変換 |
| 積載 | 数値（トン） | 積載係数に変換 |
| 停止階 | 数値（整数） | |
| 昇降行程 | 数値（メートル） | |

### 11.3 計算ロジック

```
機種係数: 標準 = 1、オーダー = 1.5
積載係数: 1トン以下 = 1、1トン超え = 2
基本個数 = 機種係数 × 積載係数 × 停止階
追加個数 = 昇降行程が 10m 超の場合、10m あたり 1 個追加
総個数 = 基本個数 + 追加個数
```

### 11.4 出力

- ブラケット個数
- 入力値一覧
- 使用した計算ルール版
- 参照した資料一覧
- 注意事項
