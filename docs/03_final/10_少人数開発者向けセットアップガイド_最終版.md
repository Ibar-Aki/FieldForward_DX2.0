## 1. 文書の目的

本書は、FieldForward DX2.0 を開発者 1〜2 名で短期間に試せるように、`setup/` 配下へ追加したセットアップ資材の使い方を整理したものである。  
目標は、共有メールボックス、SharePoint、サンプルマスタ、Office Scripts までを先に整え、Power Automate の実装へすぐ入れる状態にすることである。

---

## 2. このガイドでできること

このガイドに沿って進めると、次の状態まで到達できる。

- 開発用共有メールボックスが作成済み
- 開発用 SharePoint サイトが作成済み
- `受付原本` `送付元許可リスト` `資料マスタ` `確認キュー` `返信結果` `計算テンプレート` が作成済み
- サンプルの送付元許可リストと資料マスタが投入済み
- Excel Online に登録する Office Scripts が手元に揃っている
- あとは Power Automate `F01` から `F07` を作れば試験を始められる

---

## 3. 対象読者

- 開発担当
- 情シス
- 技術部で PoC を回す担当者

---

## 4. 前提条件

### 4.1 必要なもの

- Microsoft 365 テナント
- Exchange Online と SharePoint Online が利用可能であること
- PowerShell 7.4.6 以上
- 少なくとも 1 名の管理者権限を持つ実行者
- PnP PowerShell 用の Entra ID アプリの Client ID

### 4.2 この構成で割り切ること

- 本番の厳密な権限制御までは自動化しない
- 開発者 1〜2 名が同じ SharePoint サイトと共有メールボックスを共同利用する前提とする
- Power Automate フロー本体は tenant 内で手作成する
- Excel 正本の完全移植はせず、PoC 必須部分だけ Office Scripts 化する

## 6. 手順

## 6.1 手順1: 設定ファイルを用意する

`setup/config/developer-sandbox.example.json` をベースに、次を実環境の値へ置き換える。

| 設定キー | 入れる値 |
| --- | --- |
| `tenant.adminUpn` | Exchange / SharePoint 管理者 UPN |
| `tenant.sharePointAdminUrl` | SharePoint 管理サイト URL |
| `tenant.pnpClientId` | PnP PowerShell 用 Entra ID アプリの Client ID |
| `environment.siteUrl` | 開発用 SharePoint サイト URL |
| `environment.owner` | 開発サイト所有者 |
| `mailbox.primarySmtpAddress` | 開発用共有メールボックスアドレス |
| `groups.*` | 実際に試す開発者のメールアドレス |

少人数開発では、`groups.operators` と `groups.developers` に同じ 1〜2 名を入れてよい。

## 6.2 手順2: 前提モジュールを入れる

次を実行する。

```powershell
pwsh -File .\setup\powershell\Install-Prerequisites.ps1
```

このスクリプトは、`PnP.PowerShell` と `ExchangeOnlineManagement` を確認し、なければインストールする。

## 6.3 手順3: 一括セットアップを実行する

次を実行する。

```powershell
pwsh -File .\setup\powershell\Bootstrap-DeveloperSandbox.ps1 `
  -ConfigPath .\setup\config\developer-sandbox.example.json
```

この 1 本で、次が順に実行される。

1. 前提モジュール確認
2. 共有メールボックス作成
3. SharePoint サイト作成
4. リスト / ライブラリ / 主要列作成
5. サンプルデータ投入
6. セットアップ結果検証

共有メールボックスを手動で先に作っている場合は、次を使う。

```powershell
pwsh -File .\setup\powershell\Bootstrap-DeveloperSandbox.ps1 `
  -ConfigPath .\setup\config\developer-sandbox.example.json `
  -SkipMailbox
```

## 6.4 手順4: Excel Online へ Office Scripts を登録する

`setup/office-scripts/` に次の 3 本を追加している。

- `setInputAndCalc.ts`
- `readOutput.ts`
- `clearWorkbookState.ts`

これらは Excel Online の自動化タブへ登録する。初期状態では `Calculator` シートと以下のセル配置を前提にしている。

| 項目 | セル |
| --- | --- |
| 機種 | `B2` |
| 積載 | `B3` |
| 停止階 | `B4` |
| 昇降行程 | `B5` |
| 案件ID | `B6` |
| ブラケット個数 | `E2` |
| ルール版 | `E3` |
| 計算日時 | `E4` |

セル位置が違う場合は、Power Automate から渡すパラメータで上書きする。

## 6.5 手順5: Power Automate フローを作る

セットアップスクリプトでは、フロー本体までは作らない。ここから先は tenant 上で以下を作る。

| フロー | 役割 |
| --- | --- |
| `F01_メール受付` | メール受信、原本保存、キュー起票 |
| `F02_案件特定` | 許可判定、案件候補確定準備 |
| `F03_自動可否判定` | 自動処理 or 確認待ち判定 |
| `F04_計算実行` | Excel / Office Scripts 実行 |
| `F05_帳票生成` | 結果帳票データ作成 |
| `F06_返信送信` | 共有メールボックスから返信 |
| `F07_失敗通知` | 異常時の通知と状態更新 |

フロー仕様そのものは、次の文書を使う。

- `docs/03_final/03_設計書_最終版.md`
- `docs/03_final/04_詳細設計書_最終版.md`
- `docs/03_final/07_PoCタスク一覧_issue形式_最終版.md`

## 6.6 手順6: 最低限の動作確認をする

最低限、次を確認する。

- テスト用送付元からメール送信できる
- `受付原本` に PDF が保存される
- `確認キュー` に行が作成される
- Excel Online 上で Office Scripts が動く
- 停止条件のケースが `確認待ち` または `失敗` に遷移する

---

## 7. 少人数開発でのおすすめ運用

- 共有メールボックスは 1 つに固定する
- SharePoint サイトも 1 つに固定する
- 開発者 2 名とも `operators` と `developers` に入れる
- 送付元許可リストは最初は開発者本人だけ登録する
- 資料マスタは 3 件程度の最小構成で始める
- 停止条件は広めに取り、自動化より安全側に倒す

---

## 8. ここまで自動化済みで、ここから先は手動

### 自動化済み

- PowerShell モジュール導入
- 共有メールボックス作成
- SharePoint サイト作成
- SharePoint リスト / ライブラリ / 主要列作成
- サンプルデータ投入
- セットアップ検証

### 手動で残るもの

- Excel 正本の内容整理
- Office Scripts の Excel Online への登録
- Power Automate フロー作成
- 結果帳票テンプレート差し替え
- 返信文面の確定

---

## 9. つまずきやすい点

| 症状 | 見る場所 | 対応 |
| --- | --- | --- |
| PnP 接続で止まる | `tenant.pnpClientId` | Entra ID アプリの Client ID を確認 |
| 共有メールボックス作成で止まる | Exchange 権限 | 管理者 UPN と権限を確認 |
| CSV 投入で止まる | サンプル CSV | ヘッダ名と列名の一致を確認 |
| Office Scripts が動かない | Excel シート名 / セル位置 | `Calculator` シートとセル番地を確認 |
| フローで計算できない | Excel 正本 | VBA 依存部が残っていないか確認 |

---

## 10. 開始ライン

最短で始めるなら、次だけでよい。

1. 設定 JSON を埋める
2. `Bootstrap-DeveloperSandbox.ps1` を実行する
3. Office Scripts を Excel Online へ登録する
4. `F01` `F02` `F03` `F04` を先に作る
5. 停止ケースは全部 `確認キュー` に逃がす

この状態で、開発者 1〜2 名が同じ環境を使って PoC 実装を始められる。
