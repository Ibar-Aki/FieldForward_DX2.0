## 1. 文書の目的

本書は、案B の PoC 実装を GitHub Issues やチケット管理へそのまま転記しやすい粒度で分解した一覧である。各 issue には、目的、完了条件、依存関係、見積り、主担当を付けている。

---

## 2. 運用ルール

- 1 issue = 1 完了責任の単位
- 見積りは `時間` で管理する
- 状態は `To Do / Doing / Review / Done` を基本とする
- 技術部レビューが必要なものは `Review` を必須とする
- PoC 範囲外の要求は別 issue として切り出す

## 4. issue 詳細

### ISSUE-01: PoC スコープと成功条件を確定する

- 目的: 対象現場、対象機種、対象資料、対象ロジックを固定し、要求膨張を防ぐ
- 完了条件:
  - PoC 対象案件、対象現場、対象機種が決まっている
  - 成功条件、撤退条件が明文化されている
  - PoC 対象外が明記されている
- 依存: なし
- 目安時間: 6
- 主担当: 技術部

### ISSUE-02: 共有メールボックスと権限を準備する

- 目的: PoC 受付用のメール経路を用意する
- 完了条件:
  - 検証用共有メールボックスが作成されている
  - 関係者のアクセス権が設定されている
  - テスト送受信が確認できている
- 依存: ISSUE-01
- 目安時間: 4
- 主担当: 情シス

### ISSUE-03: SharePoint サイトと基本オブジェクトを作成する

- 目的: 原本、マスタ、キュー、返信結果の格納基盤を作る
- 完了条件:
  - `受付原本`、`返信結果`、`計算テンプレート` が作成済み
  - `送付元案件マスタ`、`資料マスタ`、`確認キュー` が作成済み
- 依存: ISSUE-01
- 目安時間: 8
- 主担当: 開発

### ISSUE-04: SharePoint 列定義、インデックス、権限を整備する

- 目的: 後工程で手戻りしないようデータ構造を固める
- 完了条件:
  - 列定義が詳細設計どおりに設定されている
  - 必須インデックスが設定されている
  - 権限ポリシーが反映されている
- 依存: ISSUE-03
- 目安時間: 8
- 主担当: 開発

### ISSUE-05: 送付元案件マスタと資料マスタの初期データを投入する

- 完了条件:

  - 資料版が特定できる状態になっている
  - 不足データが一覧化されている
- 依存: ISSUE-04

### ISSUE-06: Excel 正本の I/O 仕様と版管理ルールを確定する

## 5. 優先順位の考え方

- `ISSUE-01` から `ISSUE-06` は前提を固める基盤系であり、後回しにしない
- `ISSUE-07` から `ISSUE-13` が PoC の実装本体である
- `ISSUE-14` 以降は品質保証と意思決定のために必須である

1. `帳票体裁の作り込み` は別 issue へ切り出し、PoC 本体に混ぜない
2. `OCR 導入` は PoC 完了後の別マイルストーンにする
3. 各 issue の完了条件は、レビュー観点まで含めて固定する
