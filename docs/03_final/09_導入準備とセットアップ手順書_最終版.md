## 1. 文書の目的

本書は、FieldForward_DX2.0 の推奨方式である案Bを、M365 上で実装し、利用開始するまでの作業を 1 から順番に説明する手順書である。  
特に、現場利用者が個別のセットアップをしなくて済むように、導入担当側で事前に何を準備し、どこまで作り込むべきかを明確にする。

---

## 2. 最初に押さえるべき前提

### 2.1 このシステムの基本思想

- 現場利用者は、登録済みメールアドレスから仕様票 PDF を送るだけで使える状態を目指す
- 現場利用者に、専用アプリの導入、ブラウザ拡張、Excel マクロ設定、Power Automate 個人設定を求めない
- セットアップ作業は、情シス、開発、技術部、運用担当が導入前に集中的に実施する
- 判定が危険なケースは無理に自動化せず、確認キューへ止める

### 2.2 現場利用者が実際にやること

導入完了後に、現場利用者がやる作業は原則として次の 3 つだけである。

1. 登録済みメールアドレスから専用共有メールボックス宛にメールを送る
2. 1 通につき 1 件の仕様票 PDF を添付する
3. 返信された結果帳票と必要資料を確認する

### 2.3 現場利用者にやらせない作業

- SharePoint サイトの閲覧設定
- 個人用 Power Automate フローの作成
- Office Scripts の設定
- Excel テンプレートの直接操作
- 個別のマクロ有効化
- 手入力前提の画面操作

| 運用担当 | 確認キュー処理、障害一次対応、再実行 |
| 受入責任者 | PoC 判定、Go / No-Go 判断 |

## 4. 導入全体の進め方

導入は、以下の順で進める。

1. PoC 対象範囲を確定する
2. 必要な業務資料とマスタを準備する
3. M365 の共通基盤を作る
4. SharePoint の保存先と管理リストを作る
5. Excel 計算テンプレートと Office Scripts を準備する
6. Power Automate フロー `F01` から `F07` を作る
7. テストを行い、停止条件と手動対応を確認する
8. 運用担当へ引き継ぎ、現場へ利用開始を案内する

---

## 5. 導入前に決めること

まず、次の項目を決める。ここが曖昧だと後戻りが大きい。

| 項目 | 決める内容 | 主担当 |
| --- | --- | --- |
| PoC 対象機能 | まず何を自動化するか。初期はブラケット個数算出に限定 | 技術部 |
| PoC 対象者 | 送付元許可リストへ最初に登録する対象者 | 技術部 |
| 専用メールアドレス | 受付と返信に使う共有メールボックス名 | 情シス |
| 使用資料 | 返却に使うマニュアル、図面、帳票テンプレート | 技術部 |
| Excel 正本 | 計算に使う Excel ブックと入出力セル仕様 | 技術部 |
| 確認担当者 | 停止時に確認キューを見る担当グループ | 技術部 / 運用 |
| 本番 / 検証分離 | どこまで環境を分けるか | 情シス / 開発 |

## 6. 作業手順

## 6.1 ステップ1: PoC 対象を確定する

### 目的

最初に対象を絞り、実装範囲を広げすぎないようにする。

### やること

- PoC 対象機能を 1 つに絞る
- 対象案件の条件を決める
- PoC 対象の送付元を 50 名程度までに絞る
- 対象資料を確定する
- 停止条件を確定する

### 完了条件

- 対象機能、対象者、対象資料、停止条件が一覧化されている

### 現場利用者に依頼すること

- なし。PoC 対象者として登録されるメールアドレスの確認のみ

## 6.2 ステップ2: 必要資料を集める

### 目的

システムが参照する正本を先に揃える。

### やること

- 既存 Excel 計算ツールの正本を確定する
- 入力セルと出力セルを一覧化する
- 返却する共通マニュアル、共通図面、現場専用図面を整理する
- 資料版の承認状態を整理する
- メール返信に入れる結果帳票の最低限レイアウトを決める

### 完了条件

- Excel 正本、資料マスタ候補、結果帳票のひな形が揃っている

### 現場利用者に依頼すること

- なし

## 6.3 ステップ3: 共有メールボックスを作る

### 目的

個人メールではなく、共通の受付窓口を作る。

### やること

- Exchange Online で共有メールボックスを作成する
- 受付用メールアドレスを決める
- 運用担当と開発担当へアクセス権を付与する
- 返信元も同じ共有メールボックスに統一する

### 推奨方針

- 個人アカウントではなく共有メールボックスを使う
- 自動返信は必ず共有メールボックス名義で送る
- 受信トレイの手動操作を減らすため、受信ルールは最小限にする

### 完了条件

- テストメールを受信できる
- 共有メールボックス名義で返信できる

### 現場利用者に依頼すること

- 専用メールアドレスを知るだけでよい

## 6.4 ステップ4: SharePoint サイトを作る

### 目的

原本、マスタ、確認キュー、返信結果を一元管理する。

### やること

- `FieldForward_DX2` サイトを作成する
- 次のライブラリ / リストを作成する

| 種別 | 名称 |
| --- | --- |
| ドキュメントライブラリ | `受付原本` |
| リスト | `送付元許可リスト` |
| リスト | `資料マスタ` |
| リスト | `確認キュー` |
| ドキュメントライブラリ | `返信結果` |
| ドキュメントライブラリ | `計算テンプレート` |

- 設計書どおりに主要列を作成する
- 検索に使う列へインデックスを設定する
- 本番用と検証用の分離方針を決める

### 完了条件

- 必要なライブラリとリストが作成済み
- 列定義とインデックス設定が完了済み

### 現場利用者に依頼すること

- なし

## 6.5 ステップ5: 権限を設定する

### 目的

現場利用者に余計な設定や閲覧操作をさせず、必要な担当だけが裏側を触れるようにする。

### やること

- 運用担当グループを作成する
- 技術部グループを作成する
- 開発グループを作成する
- 各リスト / ライブラリに権限を割り当てる

### 推奨権限

| 対象 | 編集できる人 |
| --- | --- |
| 受付原本 | 運用担当、技術部 |
| 送付元許可リスト | 技術部 |
| 資料マスタ | 技術部 |
| 確認キュー | 運用担当、技術部 |
| 返信結果 | 運用担当、技術部 |
| 計算テンプレート | 技術部、開発 |

### 完了条件

- 個人ではなくグループ単位で権限設定されている
- フロー所有者変更時も運用継続できる状態になっている

### 現場利用者に依頼すること

- なし

## 6.6 ステップ6: 送付元許可リストと資料マスタを投入する

### 目的

現場利用者がメールを送るだけで処理できるように、裏側の判定材料を先に登録する。

### やること

- `送付元許可リスト` に PoC 対象者を登録する
- `受付許可`、`自動返信可否`、`確認担当者` を設定する
- `資料マスタ` に返却対象資料、資料版、承認状態を登録する
- 資料版未確定のものは本番返却対象に含めない

### 完了条件

- PoC 対象者の送付元が登録済み
- 返却対象資料の版が確定済み

### 現場利用者に依頼すること

- 自分が使うメールアドレスが登録対象か確認するだけでよい

## 6.7 ステップ7: Excel テンプレートと Office Scripts を準備する

### 目的

既存 Excel の強みを活かしつつ、個人 PC 依存をなくす。

### やること

- `計算テンプレート` ライブラリへ Excel 正本を配置する
- 入力セル、出力セル、初期化手順を整理する
- Office Scripts として以下を作る

| スクリプト名 | 役割 |
| --- | --- |
| `setInputAndCalc` | 入力値設定と再計算 |
| `readOutput` | 計算結果読込 |
| `clearWorkbookState` | 状態初期化 |

- VBA をそのまま使わず、PoC 必須部分だけ切り出して移植する

### 完了条件

- ブラウザから Excel が開ける
- Office Scripts を手動実行して期待値が返る

### 現場利用者に依頼すること

- なし

## 6.8 ステップ8: Power Automate フローを作る

### 目的

メール受付から返信までの流れを共通基盤化する。

### 作るフロー

| フロー名 | 役割 |
| --- | --- |
| `F01_メール受付` | メール受信、原本保存、キュー起票 |
| `F02_案件特定` | 送付元許可判定、案件候補確定準備 |
| `F03_自動可否判定` | 自動実行か確認待ちかを判定 |
| `F04_計算実行` | Excel / Office Scripts による計算 |
| `F05_帳票生成` | 結果帳票データ作成 |
| `F06_返信送信` | 共有メールボックスから返信 |
| `F07_失敗通知` | エラー通知、状態更新 |

### 実装時の重要ポイント

- 個人用フローではなく、組織管理前提のフローにする
- 接続参照や所有者が個人依存にならないようにする
- 停止条件を無理に通さない
- `F01` から `F03` を先に固める
- `F04` は Excel 複製、Script 実行、結果読込の順で実装する

### 完了条件

- テストメール 1 件で `F01` から `F07` が通る
- 停止条件ケースで `確認キュー` に止まる

### 現場利用者に依頼すること

- なし

## 6.9 ステップ9: 結果帳票と返信文面を作る

### 目的

現場利用者が説明なしで読める返却物にする。

### やること

- 結果帳票に以下を必ず載せる

| 必須項目 | 内容 |
| --- | --- |
| 正式指示値 | 計算結果 |
| 適用ルール版 | どの版を使ったか |
| 使用資料版 | どの資料版を参照したか |
| 注意事項 | 例外条件や確認事項 |
| 問い合わせ先 | 技術部または運用窓口 |

- 返信メール文面を決める
- 容量超過時はリンク返信または分割送信に切り替える

### 完了条件

- 初見の利用者でも意味が分かる帳票と返信メールができている

### 現場利用者に依頼すること

- なし

## 6.10 ステップ10: テストする

### 目的

利用開始前に、正常系と停止系の両方が想定どおりか確認する。

### 最低限やるテスト

- 正常な 1 メール 1 PDF が自動返信される
- 未登録送付元が停止する
- 複数 PDF 添付が停止する
- 重複送信が停止する
- 計算失敗時に確認キューへ遷移する
- 資料版未確定時に返信しない
- 手動フォールバックに切り替えられる

### PoC 合格目安

- PoC 対象サンプル 50 件中、正常系 45 件以上成功
- 停止対象ケース 100% が確認キューへ遷移
- 誤返信 0 件

### 現場利用者に依頼すること

- 代表利用者数名に試用してもらい、返信帳票の分かりやすさを確認する

## 6.11 ステップ11: 引継ぎと利用開始

### 目的

導入後に、現場利用者が何も設定せず使える状態まで持っていく。

### やること

- 運用担当へ確認キュー操作を引き継ぐ
- 障害時の連絡先と一次対応手順を決める
- 現場向け案内文を配布する
- 利用開始日を決める

### 現場向け案内に書く内容

- 専用メールアドレス
- 1 通につき 1 PDF で送ること
- 件名ルールがあるならその書き方
- 返信が来ない場合の問い合わせ先
- 対応外ケースの扱い

### 完了条件

- 運用担当が単独で再実行と手動切替を行える
- 現場利用者へ案内が出せる

## 7. 利用開始後に現場利用者へ伝える簡易手順

現場利用者向けには、以下だけ伝えればよい。

1. 登録済みメールアドレスから専用アドレスへ送る
2. 仕様票 PDF を 1 通に 1 件だけ添付する
3. 返信メールの結果帳票を確認する
4. 返信が来ない、または差戻しになった場合は問い合わせ先へ連絡する

## 8. 利用者のセッティング作業を減らすための実装ルール

導入担当は、次のルールを守る。

- 個人メールアドレスではなく共有メールボックスを使う
- 個人フローではなく組織運用可能なフローにする
- 権限は個人ではなくグループに付与する
- 現場利用者へ SharePoint 操作を要求しない
- 現場利用者へ Excel やマクロの直接操作を要求しない
- 確認が必要なケースは確認キューへ止め、現場利用者へ追加入力を求めない
- 送付元許可リストと資料マスタは導入前に十分整備する

## 10. よくある詰まりどころ

| 詰まりどころ | 起きやすい原因 | 対応 |
| --- | --- | --- |
| メールは来るがフローが動かない | 共有メールボックス設定不足 | 接続先、トリガー対象、権限を確認 |
| 受付後に止まる | 許可リスト未登録 | 送付元許可リストを確認 |
| 計算できない | VBA 依存が残っている | PoC 必須部分を Office Scripts へ切り出す |
| 返信できない | 添付容量超過 | リンク返信または分割送信へ切替 |
| 現場から問い合わせが増える | 結果帳票が分かりにくい | 帳票文言と問い合わせ先を見直す |
| 運用担当しか直せない | 個人依存で設定した | グループ権限と共有所有へ変更 |

## 11. まず最小で始めるなら

最小構成で始める場合は、次だけやればよい。

1. 共有メールボックスを 1 つ作る
2. SharePoint に `受付原本` `送付元許可リスト` `確認キュー` `計算テンプレート` を作る
3. PoC 対象 50 名の送付元を登録する
4. ブラケット個数算出だけ Office Scripts 化する
5. `F01` `F02` `F03` `F04` `F06` を先に動かす
6. 停止ケースはすべて確認キューへ回す

この順なら、現場利用者側の準備は最小のまま PoC を開始できる。
